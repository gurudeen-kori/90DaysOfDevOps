# Day 51 – Kubernetes Manifests and Your First Pods

## Task 1: Create Your First Pod (Nginx)

Created an Nginx Pod using a Kubernetes manifest.

### nginx-pod.yaml

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: nginx-pod
  labels:
    app: nginx
spec:
  containers:
  - name: nginx
    image: nginx:latest
    ports:
    - containerPort: 80
```

### Commands Executed

```bash
kubectl apply -f nginx-pod.yaml
kubectl get pods
kubectl get pods -o wide
kubectl describe pod nginx-pod
kubectl logs nginx-pod
kubectl exec -it nginx-pod -- /bin/bash
curl localhost:80
```

### Result

* Pod created successfully.
* Pod status changed to Running.
* Accessed container shell.
* Verified Nginx welcome page from inside the container.

## Task 2: Create a Custom Pod (BusyBox)

Created a BusyBox Pod with a custom command.

### busybox-pod.yaml

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: busybox-pod
  labels:
    app: busybox
    environment: dev
spec:
  containers:
  - name: busybox
    image: busybox:latest
    command:
    - sh
    - -c
    - echo Hello from BusyBox && sleep 3600
```

### Commands Executed

```bash
kubectl apply -f busybox-pod.yaml
kubectl get pods
kubectl logs busybox-pod
```

### Result

* BusyBox Pod created successfully.
* Logs displayed:

```text
Hello from BusyBox
```

* The sleep command kept the container running.

## Task 3: Imperative vs Declarative

### Imperative Method

```bash
kubectl run redis-pod --image=redis:latest
```

### Check Pod

```bash
kubectl get pods
```

### Generate YAML

```bash
kubectl get pod redis-pod -o yaml
```

### Dry Run YAML Generation

```bash
kubectl run test-pod --image=nginx --dry-run=client -o yaml
```

### Result

* Created Pod using imperative command.
* Compared generated YAML with handwritten manifests.
* Observed additional metadata generated by Kubernetes.

## Task 4: Validate Before Applying

### Client-side Validation

```bash
kubectl apply -f nginx-pod.yaml --dry-run=client
```

### Server-side Validation

```bash
kubectl apply -f nginx-pod.yaml --dry-run=server
```

### Validation Error Test

Removed the image field and validated again.

### Result

```text
spec.containers[0].image: Required value
```

## Task 5: Pod Labels and Filtering

### Show Labels

```bash
kubectl get pods --show-labels
```

### Filter Pods

```bash
kubectl get pods -l app=nginx
kubectl get pods -l environment=dev
```

### Add Label

```bash
kubectl label pod nginx-pod environment=production
```

### Remove Label

```bash
kubectl label pod nginx-pod environment-
```

### Third Pod Manifest

apiVersion: v1
kind: Pod

### Result

* Successfully filtered Pods using labels.
* Added and removed labels dynamically.
* Created a Pod with multiple labels.

kubectl get pods
